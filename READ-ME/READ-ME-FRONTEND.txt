ARXITEKTONIKH:

 Το κεντρικο design pattern ειναι αυτο του Model <---> View, οπου η προβολη καθε σελιδας συνδεεται διμερως με ενα συνολο μεταβλητων,
 επιτρεποντας την ροη δεδομενων μεταξυ τους

 Η υλοποιηση μιας σελιδας διασπαται λειτουργικα σε επιμερους τμηματα, τα οποια ειναι τα:

 * Components
 * Services
 * Guards, και
 * Interceptors


* Components:

 Ως component οριζεται οποιαδηποτε κλαση που διαθετει ενα συνολο μεταβλητων οι οποιες ειναι συνδεδεμενες με την οπτικη αναπαρασταση αυτου
 

* Services:

Τα services εξυπηρετουν τα components που τα καλουνε, παρεχοντας λειτουργικοτητες οπως:

 * αποστολη αιτηματων εγγραφης, επανεγγραφης, διαγραφης και αναγνωσης δεδομενων που φιλοξενουνται στον server
 
 * αποστολη και ληψη δεδομενων μεταξυ των components

 * υπηρεσιες ταυτοποιησης και εξουσιοδοτησης χρηστη
 
 * διαφορες συναρτησεις μετατροπης δεδομενων

* Guards:

 Τα guards εξυπηρετουν στον ελεγχο υπαρξης των απαραιτητων δικαιωματων πριν ο χρηστης επισκφθει ενα συγκεκριμενο endpoint


* Interceptors:

 Ο ρολος ενος interceptor ειναι η ληψη ενος αιτηματος ή ανταποκρισης προς ή απο τον server αντιστοιχα, με σκοπο την αναγνωση, την επεξεργασια και τελικα την προωθηση του
 σε καποιον αλλον interceptor, component ή στον server


ΠΡΟΚΛΗΣΕΙΣ:

 * Παροχη feedback στο χρηστη:

 - Οταν ο χρηστης εκτελει καποια ενεργεια εντος της τρεχουσας σελιδας, πρεπει να ειναι σε θεση να γνωριζει τι αντικτυπο ειχε η εκαστοτε ενεργεια ειτε στα δεδομενα που προβαλλονται
   ή στα προσωπικα του δεδομενα, που βρισκονται αποθηκευμενα στο server. Η λυση του προβληματος αυτου εγινε με τη χρηση προειδοποιητικων dialogs και animations με σκοπο την ληψη προσοχης
   του χρηστη

 * Δυνατοτητα παροχης ενος δυναμικου layout:

 - Η προβολη μιας σελιδας οφειλει να προσαρμοζεται οπτικα στα δεδομενα που φιλοξενει, ανεξαρτητως των τιμων των δεδομενων. Το προβλημα αυτο λυθηκε (εν μερει) με τη χρηση του Flexbox
   της CSS που επιτρεπει την ευκολη ευθυγραμμιση και προβολη οπτικου περιεχομενου
